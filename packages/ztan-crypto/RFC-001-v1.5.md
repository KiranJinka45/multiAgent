# ZTAN-RFC-001 v1.5: Canonical Encoding & Threshold Signing

## 1. Introduction
This document defines the canonical binary representation and signing process for the Zero-Trust Audit Network (ZTAN). It ensures that multiple runtimes (Node.js, Python, etc.) arrive at the same byte-perfect hash for any given audit event.

## 2. Canonical Encoding Rules (CER)
### 2.1 Integer Encoding
- All integers MUST be encoded as 8-byte (64-bit) Big-Endian unsigned integers.

### 2.2 String Encoding
- All strings MUST be UTF-8 encoded.
- Strings MUST be prefixed with their length as a 4-byte (32-bit) Big-Endian unsigned integer.

### 2.3 List Encoding
- Lists MUST be prefixed with the number of elements as a 4-byte Big-Endian integer.
- List items MUST be sorted lexicographically by their byte representation.

## 3. Cryptographic Primitives
- **Elliptic Curve:** BLS12-381.
- **Hash Function:** SHA-256.
- **Domain Separation Tag (DST):** `ZTAN-V1-BLS12381-SHA256-SIGNING-DST`.

## 4. Signing Process
1. **Preimage Creation:** `SHA256(version || messageId || nodeId || payload || timestamp)`.
2. **Threshold Aggregation:** Partial signatures are aggregated using Lagrange interpolation over G1.
3. **Verification:** The aggregate signature is verified against the Master Public Key (MPK).
